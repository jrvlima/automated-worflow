DevOps

  • Operation is not it’s own silo, but a liaison between operations and developers
  • Ops been at inceptions, showcases and retros
  • As CD requires machines to be provisioning programatically we will drive a much earlier awareness of operations concerns
  • Once in a while the idea of Dev works in ops can carry pagers
  • The goal is developers create more deployable software

Destroy Works of Art through Infrastructure as Code

Infrastructure as code

  • Refactoring
  • Testing
  • Build Pipeline
  • Self Documenting
  • Infrastructure = environments and supporting services(networking, vcs, storage, mails, DNS …)
  • Desidere state specified in version control
  • Autonomic(self-corrects to desired state) - ex: a node state version can be set in the server
  • State should be known through monitoring

Configuration Management tools as Chef, Puppet, Ansible and etc, provide us ways to manage many systems, manage configuration, enforce consistency, monitor and treat infrastructure as code
